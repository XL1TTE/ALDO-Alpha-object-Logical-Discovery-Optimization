# Отчет: Оптимизационная модель поиска логических закономерностей (Метод Альфа-объекта)

## Определение Альфа-объекта

В контексте интеллектуального анализа данных и распознавания образов **Альфа-объект** представляет собой опорное наблюдение из обучающей выборки, используемое в качестве базовой точки для построения локальной закономерности (паттерна). Метод предполагает выбор конкретного объекта одного из классов и построение вокруг него гиперпараллелепипеда в пространстве признаков, который удовлетворяет заданным критериям оптимальности.

Критерии построения модели на основе Альфа-объекта:
1. **Условие покрытия:** Построенная область (паттерн) должна обязательно содержать выбранный Альфа-объект.
2. **Максимизация покрытия:** Модель стремится охватить максимально возможное количество объектов того же класса, что и Альфа-объект.
3. **Условие чистоты:** Модель накладывает строгие ограничения, исключающие попадание в область паттерна любых объектов противоположного класса.

---

## Цель и область применения

Данный подход применяется для извлечения интерпретируемых логических правил из массивов данных. В отличие от стохастических методов или глубокого обучения, результат метода Альфа-объекта представляется в виде системы предикатов над признаками (например, $x_j \in [a_j, b_j]$), что позволяет:
* Обеспечить полную прозрачность процесса классификации.
* Выявить значимые диапазоны значений признаков для каждого класса.
* Сформировать базу знаний для экспертных систем.

---

## Входные и выходные параметры

### Входные данные
Алгоритм принимает на вход структурированные табличные данные, содержащие:
* **Множество признаков (Атрибутов):** Вектор числовых характеристик произвольной размерности $n$.
* **Метку класса:** Бинарный индикатор принадлежности объекта к положительному или отрицательному классу.

### Выходные данные
Результатом работы алгоритма является формализованная математическая модель нелинейной оптимизации. Модель описывается целевой функцией (максимизация суммы индикаторных функций) и системой ограничений, задающих границы гиперпараллелепипеда $[a_j, b_j]$ в пространстве признаков.

---

## Описание программного решения

Разработанный программный комплекс автоматизирует процесс составления оптимизационных моделей для каждого объекта выборки. Основные функциональные возможности включают:

1. **Динамическая размерность:** Программное обеспечение автоматически адаптируется к количеству признаков в наборе данных, формируя соответствующие переменные и ограничения для $n$-мерного пространства.
2. **Многокритериальный вывод:** Поддержка различных форматов представления моделей, включая текстовый формат для технического анализа и формат Markdown с использованием нотации LaTeX для академической отчетности.
3. **Архитектурная гибкость:** Реализация на основе паттерна "Стратегия" позволяет расширять способы форматирования данных без изменения основного алгоритма генерации.
4. **Интерфейс пользователя:** Наличие интерактивного режима с использованием нативных диалоговых окон ОС и автоматизированного интерфейса командной строки (CLI) для пакетной обработки.

## Пример формализации модели

### Описание входного набора данных
Рассмотрим пример набора данных с тремя признаками ($n=3$):

| Индекс | X1 | X2 | X3 | Класс |
|:---:|:---:|:---:|:---:|:---:|
| 0 | 1.80 | 75.0 | 36.6 | 1 |
| 1 | 1.50 | 90.0 | 38.5 | 0 |

### Формируемая оптимизационная задача (для Альфа-объекта №0)
При выборе объекта №0 в качестве опорного, система генерирует следующую модель:

**Переменные решения:** $a_j, b_j$ для $j \in \{1, 2, 3\}$.

**Целевая функция:**
$$\max Z = \sum_{i \in S^+} \mathbb{I}(x_{i1} \in [a_1, b_1] \land x_{i2} \in [a_2, b_2] \land x_{i3} \in [a_3, b_3])$$

**Основные ограничения:**
1. Ограничения Альфа-объекта: $a_1 \le 1.80 \le b_1, \ a_2 \le 75.0 \le b_2, \ a_3 \le 36.6 \le b_3$.
2. Ограничения на отрицательные объекты: для объекта №1 индикаторная функция покрытия должна быть равна 0.
